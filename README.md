## 개요
- **프로젝트 성격**: KH정보교육원 국비 팀 프로젝트(3인)
- **주요 역할**: UI/UX 퍼블리싱(시약 현황 관리, 사용 등록, 승인 워크플로우)
- **작업 기간**: 2025.06 ~ 2025.07(5주)

## 기술 스택
### **Frontend**

- Library: jQuery, Bootstrap 5, JSTL
- View: JSP

---

### **Backend & Database**
- Framework: Spring Boot 3.5.4, Spring Security, MyBatis 3.0.3
- Language: Java 21
- DB/Tools: MySQL, Maven, GitHub

---

## 핵심 구현 및 퍼블리싱 포인트
- **웹 표준 및 접근성 준수**: 스크린 리더 사용자를 고려하여 제목 구조(h1 ~ h3)를 설계하고, 이미지 요소에 alt 속성을 빠짐없이 적용했습니다.
- **흐름의 연속성 강조**: 시약 승인 절차 시, 페이지 전환을 최소화하기 위해 모달(Modal) UI를 활용하여 사용자 동선의 뎁스(Depth)를 최적화했습니다.
- **상태 유지 필터 및 동적 데이터 매핑**: input 태그 내 JSTL 삼항 연산자를 활용하여 검색 파라미터에 따라 라디오 버튼이 자동 체크되도록 화면 상태를 제어했습니다.  
	페이지 리로드 시에도 검색 조건이 유지되도록 설계했으며, 필터링된 DB 데이터를 테이블 레이아웃에 정확하게 매핑하여 데이터 조회 편의성을 높였습니다.

---

## 트러블 슈팅
- **[현상]** 사용자 입력 데이터(제목 등)가 예상보다 길어질 경우, 레이아웃이 무너지고 텍스트가 겹치는 문제 발생.  
- **[원인]** 고정된 높이의 박스 안에서 텍스트 길이에 대한 예외 처리 부족.  
- **[해결]** CSS의 `text-overflow: ellipsis;` 속성을 활용하여 말줄임표 처리를 적용, 데이터 길이에 관계없이 일정한 레이아웃을 유지하도록 설계했습니다.  
- **[배운점]** 실데이터가 들어왔을 때 발생할 수 있는 변수를 고려한 예외 처리의 중요성을 깨달았습니다.
